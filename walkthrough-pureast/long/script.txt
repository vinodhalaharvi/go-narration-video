[[title:PureAST: Functorial AST for LLM Context]]
[[file:node.go line:11]] When you give a large Go codebase to an LLM, the context window fills up fast. Most of what you send is noise. PureAST extracts only the relevant signatures, types, and dependencies. Here is the central type: Dependencies — a record of sets for types, functions, structs, interfaces, imports, constants, and variables.
[[line:22]] NewDependencies returns the empty value. This is the identity element of our monoid. Composition starts here, with nothing.
[[line:35]] We declare DependencyMonoid. A monoid is a type with an identity element and an associative combine operation. That is all we need.
[[line:47]] Combine merges two dependency sets by combining each field with the underlying set monoid. Because sets compose under union, Dependencies compose under union. The math gives you correctness for free.
[[file:visitor.go line:9]] Now the second key abstraction. A Visitor is a function from Dependencies to Dependencies. An endomorphism. It takes accumulated dependencies and produces new accumulated dependencies.
[[line:19]] Two visitors compose by running them in sequence. ComposeVisitors is just function composition. Because Visitor is an endomorphism, composing visitors is also a visitor. The category closes.
[[file:extract.go line:14]] Here in the extract package, the visitor type is the same shape. We use an EndoMonoid from PureKernels for composition. The extraction algorithm becomes a fold over the AST that emits visitors and combines them.
[[line:67]] ExtractType walks a Go expression and returns a Visitor that adds the right dependencies. It never mutates anything. It returns a function. The actual work happens later, when you apply that function to a starting Dependencies value.
[[line:503]] ExtractFuncDecl extracts a function declaration. It composes smaller visitors — for the receiver, the parameters, the return types, the body — into one visitor. The whole AST traversal is built from these small, composable pieces.
[[file:directory.go line:85]] FileNodeMonoid lets us combine extracted files. Empty is the empty file node. Combine merges two file nodes by combining their dependency sets and concatenating their declarations.
[[line:147]] Now the payoff for purity. ExtractDirectoryConcurrentBatched walks a whole directory, parses every Go file in parallel, and combines the results with the monoid. Because each step is pure, parallelism is free — there is nothing to synchronize.
[[file:dump.go line:31]] Finally, the application. The dump command is the LLM-context flagship. Given a package, it emits every top-level symbol — structs, interfaces, type aliases, functions — in compact form.
[[line:53]] NewDumpCommand wires it up. The default output is signatures only. Bodies are stripped. This is the single biggest token compression win — a five-thousand-line package collapses to a few hundred lines of "what exists, what it returns, what it satisfies."
[[line:300]] renderDump produces the final text. You can pipe this directly into a Claude prompt, or fit it to a token budget. The output is always syntactically complete Go, because the pipeline is symbol-aware. It drops whole declarations rather than slicing through one. That is PureAST.
