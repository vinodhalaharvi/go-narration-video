In this short, we will build a generic Fold function that walks any tree. Then we will define predicates as plain functions, compose them with And, Or, and Not, and watch our queries read like English.
[[pause:0.6]]
[[file:fold.go reveal lines:1-10]] We start with the data. A node has a name, a number, and any children that are themselves nodes. Recursive type, three fields.
[[pause:0.5]]
[[reveal lines:12-21]] Now the most important function in this video. Fold takes a tree, an initial value, and a folding function. It applies the function to the current node, then recurses into every child, threading the accumulator through. The accumulator carries state across the whole tree. The folding function decides how to update that state at each node. This single shape replaces every recursive walk you ever wrote. Counting nodes? Fold. Summing values? Fold. Collecting names? Fold. Building any data structure from a tree? Fold.
[[pause:1.0]]
[[reveal lines:23-23]] Next idea. A predicate is just a function from node to bool. Not an interface. Just a function.
[[pause:0.4]]
[[reveal lines:29-37]] Primitives. Match by name. Match by a minimum number. Check if a node is a leaf. One line each.
[[pause:0.5]]
[[reveal lines:43-45]] Combinators. Build And, Or, and Not. Three lines, every Boolean query.
[[pause:0.4]]
[[reveal lines:48-56]] Find ties it together. Fold the tree, append every match. Notice that Find is just Fold plus a predicate.
[[pause:0.7]]
[[reveal lines:69-79]] Now let us see it work. Here is a sample tree. Three branches. Some leaves. A mix of names and numbers.

[[reveal lines:83-87]] First query. Find by name returns the alpha branch and its leaf. Two matches.
